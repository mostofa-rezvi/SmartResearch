# /gsd-secure-phase

**Category:** 4. Quality Assurance & Review  
**Phase:** After security-critical phases

## Description

Retroactively verify threat mitigations for a completed phase.

## When to Use

- After auth system, API endpoints, or any phase handling user data
- Security-focused audit: token storage, CSRF, XSS, injection
- OAuth callback validation, rate limiting verification

## Command

```
/gsd-secure-phase
```
