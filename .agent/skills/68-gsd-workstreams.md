# /gsd-workstreams

**Category:** 3. Execution & Building  
**Phase:** Parallel development

## Description

Manage parallel workstreams — list, create, switch, status, progress, complete, and resume.

## When to Use

- When running backend + ML service + frontend in parallel
- Switch context between workstreams cleanly
- Track progress per workstream independently

## Commands

```
/gsd-workstreams list
/gsd-workstreams create "ML Service Build"
/gsd-workstreams switch <id>
/gsd-workstreams status
/gsd-workstreams complete
```
