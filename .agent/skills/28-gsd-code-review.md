# /gsd-code-review

**Category:** 4. Quality Assurance & Review  
**Phase:** After each phase execution

## Description

Review source files changed during a phase for bugs, security issues, and code quality problems.

## When to Use

- After each phase execution
- Catches SQL injection, missing auth checks, memory leaks
- Produces REVIEW.md with findings and severity ratings

## Command

```
/gsd-code-review
```
