# /gsd-insert-phase

**Category:** 2. Phase Planning & Discussion  
**Phase:** Urgent mid-sprint work

## Description

Insert urgent work as decimal phase (e.g., 72.1) between existing phases.

## When to Use

- Hot-fix or critical dependency discovered mid-sprint
- Example: Redis connection pooling issue blocking Day 5
- Urgent security patch needed between planned phases

## Command

```
/gsd-insert-phase
```
