# PLAN 08-02 — Wave 2: PageRank & TrustRank

## Objective
Implement the daily PageRank job to calculate researcher impact scores.

## Context & Constraints
- **Library**: Neo4j GDS.
- **Service**: Python ML Service (Orchestrator).
- **Trigger**: Daily cron or manual trigger.

## Tasks

### 8.4 GDS Graph Projection
- **Action**: Create `ml-service/jobs/graph_rank.py`.
- **Logic**: 
    1. Create/Refresh a named graph projection in GDS containing `Researcher` nodes and all trust edges.
- **Verification**: `CALL gds.graph.list()` shows the projection.

### 8.5 PageRank Execution
- **Action**: Add `calculate_impact_scores()` to `graph_rank.py`.
- **Logic**: 
    1. Execute `gds.pageRank.write` on the projection.
    2. Normalize scores to a 0-100 range for UI consistency.
- **Verification**: `impactScore` property is populated on Researcher nodes.

### 8.6 Top Contributor Badge
- **Action**: Implement post-PageRank logic.
- **Logic**: 
    1. Find researchers in the top 5th percentile of impact scores.
    2. Apply `isTopContributor: true` property.
- **Verification**: Cypher query confirms top nodes have the badge property.

## Verification
- End-to-end: Add 10 dummy researchers with random supports/citations, run PageRank, verify scores are non-zero.
