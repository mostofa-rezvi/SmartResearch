# PLAN 08-01 — Wave 1: Graph Schemas & Traversal

## Objective
Update the Neo4j schema with new trust relationships and implement the co-authorship traversal query.

## Context & Constraints
- **Relationships**: `ENDORSES`, `CITES`.
- **Query**: 2nd-degree collaborator discovery.
- **Service**: Node.js Backend.

## Tasks

### 8.1 Schema Update: Trust Edges
- **Action**: Update `backend/trust-graph-schema.cypher`.
- **Logic**: 
    1. Define `ENDORSES` relationship with `weight` and `timestamp`.
    2. Define `CITES` relationship (Paper-to-Paper).
- **Verification**: `CALL db.relationshipTypes()` shows the new types.

### 8.2 Collaborator Discovery Query
- **Action**: Create `backend/src/services/discoveryService.js`.
- **Logic**: 
    1. Implement `getSuggestedCollaborators(userId)`.
    2. Use the 2nd-degree co-authorship Cypher query.
    3. Filter out users the researcher is already connected to.
- **Verification**: Query returns valid researcher nodes in Neo4j browser.

### 8.3 Institutional Badge Sync
- **Action**: Update `backend/src/workers/graphSync.worker.js`.
- **Logic**: 
    1. Listen for user profile updates.
    2. Sync `AFFILIATED_WITH` relationship to Neo4j.
    3. Set `isVerifiedInstitutional: true` on the Researcher node if the institution exists.
- **Verification**: Node properties updated in Neo4j after Postgres change.

## Verification
- Unit test for discovery query with a mock 3-node chain.
