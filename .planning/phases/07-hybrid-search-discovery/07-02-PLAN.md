# PLAN 07-02 — Wave 2: API & Integration

## Objective
Expose the recommendation engine via REST and integrate it with the Node.js backend and Redis cache.

## Context & Constraints
- **Endpoint**: `POST /recommendations/{userId}`
- **Cache**: Redis (TTL 1hr)
- **Backend**: Node.js Proxy

## Tasks

### 7.4 Recommendations API
- **Action**: Update `ml-service/main.py`.
- **Logic**: 
    1. Add `POST /recommendations/{userId}`.
    2. Orchestrate: Call ES for CBF, call `cf_engine` for CF, then `rrf_merge`.
- **Verification**: `curl` returns a JSON list of recommended IDs.

### 7.5 Background Refresh Task
- **Action**: Create `ml-service/jobs/refresh_matrix.py`.
- **Logic**: Periodically rebuild the CF similarity matrix every 6 hours to keep recommendations fresh.
- **Verification**: Log check showing "Similarity matrix rebuilt".

### 7.6 Backend Proxy & Hydration
- **Action**: Create `backend/src/services/recommendationService.js`.
- **Logic**: 
    1. Check Redis for `rec:v1:{userId}`.
    2. If miss, call ML Service.
    3. Hydrate IDs into full Profile/Paper objects from Postgres.
- **Verification**: Frontend `/api/recommendations` returns full objects, not just IDs.

## Verification
- End-to-end integration test: User with 5 interactions sees "Suggested for You" list on dashboard.
