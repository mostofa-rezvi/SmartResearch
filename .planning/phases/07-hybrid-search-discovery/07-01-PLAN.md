# PLAN 07-01 — Wave 1: CF Engine & RRF Scorer

## Objective
Implement the core mathematical engine for recommendations in the Python ML service.

## Context & Constraints
- **Model**: User-User Similarity on interaction weights.
- **Merging**: RRF (k=60).
- **Service**: `ml-service/`.

## Tasks

### 7.1 Interaction Matrix Builder
- **Action**: Create `ml-service/recommender/matrix_builder.py`.
- **Logic**: 
    1. Fetch `saved_papers`, `votes`, and `group_members` from PostgreSQL.
    2. Fetch `SUPPORTS` from Neo4j.
    3. Build a sparse CSR matrix where rows = users, cols = items/authors.
- **Verification**: `matrix.shape` matches total users/items in DB.

### 7.2 Hybrid RRF Scorer
- **Action**: Create `ml-service/recommender/scorer.py`.
- **Logic**: 
    1. Function `rrf_merge(cbf_list, cf_list, k=60)`.
    2. Accepts two lists of `(id, score)` and returns a single list of `(id, rrf_score)`.
- **Verification**: Unit test with overlapping and non-overlapping lists.

### 7.3 Collaborative Filtering (CF) Predictor
- **Action**: Create `ml-service/recommender/cf_engine.py`.
- **Logic**: 
    1. `get_similar_users(userId, Top-N)`.
    2. `get_recommendations_from_peers(userId)`.
- **Verification**: Returns non-empty lists for active users.
