# PLAN 09-01 — Wave 1: Mappings & Vectorization

## Objective
Configure Elasticsearch indices for kNN search and integrate query vectorization into the discovery service.

## Context & Constraints
- **Vector Dim**: 768.
- **Index Type**: HNSW.

## Tasks

### 9.1 Mapping Setup Script
- **Action**: Create `scripts/setup_es_indices_knn.py`.
- **Logic**: 
    1. Create/Update indices `users`, `papers`, `projects`.
    2. Add `embedding` field with `type: dense_vector`, `dims: 768`, `index: true`, `similarity: cosine`.
- **Verification**: `GET /_mapping` confirms the field.

### 9.2 Backend Query Vectorizer
- **Action**: Update `backend/src/services/discovery.service.js`.
- **Logic**: 
    1. Add `vectorizeQuery(queryText)` method.
    2. Calls ML Service `POST /embed`.
- **Verification**: Logs show query vector received from ML service.

### 9.3 Unified Search Service (Base)
- **Action**: Implement the shell of the new `hybridSearch` method.
- **Verification**: Returns raw ES hits from kNN query only.

## Verification
- Run setup script and verify indices are kNN-ready.
