# Phase 14: Publication & Forum Logic - Plan 2

<must_haves>
- Forum posts must support threaded replies and upvoting.
- Forum feed must sort by TrustRank weight.
- TrustRank threshold gating must automatically filter out spam.
</must_haves>

<step>
<description>
Implement the core Forum Service with TrustRank weighting.
</description>
<action>
Create `src/services/forum-service.ts`.
Export a class `ForumService` with `getThreads()`, `createThread()`, `addReply()`, and `upvote()`.
In `getThreads()`, apply sorting logic that multiplies the upvote count by the author's TrustRank score.
Create endpoints `src/app/api/forum/route.ts` and `src/app/api/forum/[threadId]/route.ts` for CRUD operations.
</action>

- src/services/forum-service.ts
- src/app/api/forum/route.ts

- `src/app/api/forum/route.ts` exports `GET` and `POST` handlers
</acceptance_criteria>
</step>

<step>
<description>
Implement TrustRank Moderation & Spam Filter.
</description>
<action>
Create `src/services/trustrank-moderation.ts`.
Export a function `evaluateSpam(content: string, authorTrustRank: number)`. If `authorTrustRank < 10` and content fails heuristics (e.g., contains too many links), return `{ isSpam: true }`.
Integrate `evaluateSpam` into `ForumService.createThread` and `addReply` to reject spam posts.
</action>
