# PLAN 10-01 — Wave 1: Schema & Workspace CRUD

## Objective
Establish the relational foundation for projects and collaborative objects.

## Tasks

### 10.1 Database Migration
- **Action**: Create `backend/migrations/collaboration_tables.sql`.
- **Logic**: 
    - Tables: `projects`, `project_members`, `milestones`, `tasks`, `collaborative_docs`.
    - Constraints: Unique members per project, valid milestone statuses.
- **Verification**: `psql` shows all tables created with correct foreign keys.

### 10.2 Workspace Service & Controller
- **Action**: Create `ProjectService.js` and `ProjectController.js`.
- **Logic**: 
    - `createProject`: Sets creator as ADMIN.
    - `getProjectSummary`: Hydrates milestones and active members.
    - `inviteMember`: Validates researcher exists.
- **Verification**: `POST /api/v1/projects` returns the project ID and default milestone.

### 10.3 Milestone State Machine
- **Action**: Create `MilestoneService.js`.
- **Logic**: 
    - Implement `updateStatus` with transition validation.
    - Statuses: `TODO` -> `IN_PROGRESS` -> `REVIEW` -> `DONE`.
- **Verification**: Unauthorized status transitions return 403 or 400.

## Verification
- CRUD operations for projects and milestones functional in Postman.
