# PLAN 10-02 — Wave 2: Real-Time Sync & Presence

## Objective
Implement the Socket.IO layer and Yjs document synchronization.

## Tasks

### 10.4 Socket.IO Rooms & Presence
- **Action**: Update `backend/src/socket/index.js` (or created if not existing).
- **Logic**: 
    - `joinProject(projectId)`: Joins the project room.
    - `updatePresence`: Broadcasts user status to the room.
- **Verification**: Socket logs show correct room membership.

### 10.5 Yjs Persistence Adapter (Postgres)
- **Action**: Create `CollaborationService.js`.
- **Logic**: 
    - `loadDocument(docId)`: Fetches `BYTEA` blob and initializes Yjs Doc.
    - `saveDocument(docId, update)`: Merges update and writes back to Postgres.
- **Verification**: Refreshing the document (server-side) maintains text state.

### 10.6 Real-Time Events Bridge
- **Action**: Connect Project Service updates to Socket.IO.
- **Logic**: When a milestone or task is updated via REST, emit a `PROJECT_UPDATED` event to the corresponding Socket.IO room.
- **Verification**: Frontend (or test client) receives real-time milestone changes.

## Verification
- Collaborative sync test: Multiple socket clients can "edit" and state is merged.
