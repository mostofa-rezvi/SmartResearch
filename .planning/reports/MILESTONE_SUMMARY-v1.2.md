## 4. Key Decisions & Learnings

### Server-side Trust Integrity
- **Decision:** Shifted all TrustRank and metadata lookups to the server-side.
- **Learning:** Trust-based systems are uniquely vulnerable to "spoofing" if any reputation logic is delegated to the frontend.

### Wave-based Infrastructure
- **Decision:** Grouped infra setup before performance benchmarking.
- **Learning:** Validating HPA behavior during k6 load tests allowed us to confirm the ML service can sustain 10k concurrent connections.
